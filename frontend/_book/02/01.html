js的解析与执行过程

1. 前言

这节将讨论的知识点有：

  • 全局预处理
  • 函数预处理

2. 全局预处理

js解析器在解析执行js脚本之前都会先进行一次预处理来初始化全局词法环境。（LexicalEnvironment）。

2.1. 问题开始

如下代码：

<script>
    var a = 10;
    function func() {
        alert(a);
        //加上这句之后，上一句代码弹出了undefined
        var a = 20;
    }
    func();
</script>

分析如下：

  1. 在解析执行脚本之前，js解析器首先需要执行预处理，初始化LexicalEnvironment对象。
  2. 只预处理通过 function 函数名 这种方式定义的函数，以及var定义的变量

  3. 经过预处理之后，LexicalEnvironment对象大致如下

     //LexicalEnvironment等于window对象
 LexicalEnvironment = {
     a: undefined
     func:函数地址
 }
 //LexicalEnvironment等于func对象
 LexicalEnvironment = {
     a: undefined
 }

不要被那个全局的变量a所误解了，其实可以理解为下面这样的一段代码

<script>
    alert(a);//undefined
    var a = 10;
</script>

2.2. 问题延伸

通过上面的小节理解了全局预处理的方式，那么看如下代码的输出.

<script>
    alert(a);//输出函数
    var a = 10;
    function a() {
        console.log("111");
    }
</script>

这里输出函数，和我们的猜想一样，是覆盖的方式。

理解：

1. 顺序预处理，此时词法环境window对象中存在有 a:undefined

2. 在进行顺序预处理，遇上函数a，这时候会替换掉原有词法环境中的a，此时应该是a:函数

那么，在看如下代码：

<script>
    alert(a);//输出函数
    function a() {
        console.log("111");
    }
    var a = 10;
</script>

根据上面的理解这里应该输出的是undefined，然而并不是这样。

说明js引擎进行预处理并不是简单的顺序执行并覆盖

理解：

1. 顺序预处理，此时window对象中包含了a:函数

2. 顺序预处理，碰到变量a,因为前面的词法环境中存在了函数a，所以直接就忽略掉了变量的a

总结：

预处理，遇上函数同名会覆盖，变量同名直接忽略

为了加深理解，在看一段代码：

<script>
    alert(a);//输出函数222
    function a() {
        console.log("111");
    }
    var a = 10;
    function a() {
        console.log("222");
    }
</script>

根据上面二段总结出的理念，我们可以很清楚的知道此时a应该输出什么了。

2.3. 问题总结

通过前面两个小节，我们来分析下面代码的输出。

<script>
    alert(a);
    alert(b);
    alert(f);
    alert(g);
    var a = 5;
    b = 6;
    alert(b);
    function f()
    {
        console.log("f");
    }
    var g = function () {
        console.log("g");
    }
    alert(g);
</script>

  1. 首先进行全局预处理，根据预处理的规则，只处理声明的函数，和var定义的变量，此时词法环境应该是这样的。

     window = {
     a: undefined,
     f: function f() {
         console.log("f");
     },
     g:undefined  //当变量处理了
 }

  2. 接着顺序执行，（虽然报错之后后面的js不会执行。）

     //a  输出undefined
 //b  报错，b is not defined
 //f  输出f函数表达式
 //g  输出undefined

接着在，a赋值为5，b初始化为6.此时：

    window = {
        a:5,
        f: function f() {
            console.log("f");
        },
        g:undefined,  //当变量处理了
        b:6
    }

后面就不说了。。。。。。

  1. 那么最终的输出顺序应该是。（虽然报错之后后面的js不会执行。）

     //a  输出undefined
 //b  报错，b is not defined
 //f  输出f函数表达式
 //g  输出undefined
 //b  输出6
 //g  输出g函数表达式

3. 函数预处理

全局预处理和函数预处理一样，只是多了个参数的预处理，看代码：

<script>
    function func(a, b)
    {
        alert(a);
        alert(b);
        var a = 100;
        function b()
        {
            console.log("bb");
        }
    }
    func(1, 2);
</script>

分析：

  1. 顺序执行预处理，此时window对象的预处理是这样的

     window = {
     func:func函数的字符串表现形式
 }

  2. 顺序解析js，func(1,2)，函数调用才会执行函数预处理，此时预处理后是

     func = {
     a: 1,
     b: 2,
     arguments
 }

  3. 继续顺序解析，func函数里面的变量a和函数b，变量忽略，函数覆盖，此时func是

     func = {
     a: 1,
     b: b函数的字符串表现形式,
     arguments
 }

  4. 预处理完成之后，顺序执行js代码，输出1和函数字符串的表现形式

4. 总结

  • js在解析执行之前都会进行预处理，才会真正的执行
  • 理解js引擎对JS的预处理和执行过程。
  • 必须知道，预处理时，变量忽略，函数覆盖。
  • 全局预处理，在顺序解析js就开始进行了
  • 函数预处理，只有在调用函数才进行
